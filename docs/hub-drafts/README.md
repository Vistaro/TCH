# Nexus Hub — draft submissions

Drafts of Bugs / FRs / Milestones that the agent could not file
directly to the Hub because the API token path isn't wired yet
(see `docs/TCH_Ross_Todo.md` Item 12).

When Item 12 is fixed, these drafts can be filed by the agent
automatically. Until then, Ross can paste each draft into the Hub
UI manually, and move the filed draft to `_archive/hub-drafts/`
with the Hub-assigned ID noted at the top.

Each draft follows the standing three-layer format:

- **Title** — one line, output-focused
- **2a. Reporter description** — original words (agent-voice here,
  but written as the reporter would say it)
- **2b. Agent-enriched CxO framing** — Outcome / What it does /
  Why it matters / If we don't do it
- **3. Technical description** — Architecture context / Proposed
  approach / Dependencies + risks / Acceptance criteria
